Дефекты в реализации StringUtils:

1. Метод: capitalize()
   - Не обрабатывает строки, начинающиеся с пробела
   - Ожидалось: " skypro" → " Skypro"
   - Реальность: " skypro" → " skypro"
   - Серьёзность: minor
   - Рекомендация: Использовать string[0].upper() + string[1:] вместо capitalize()

2. Метод: trim()
   - Удаляет только обычные пробелы (U+0020), но не другие whitespace-символы
   - Ожидалось: "\t\nskypro" → "skypro"
   - Реальность: "\t\nskypro" → "\t\nskypro"
   - Серьёзность: major
   - Рекомендация: Использовать string.lstrip() или расширить логику удаления

3. Метод: contains()
   - Реализация через index() избыточна и медленнее простого 'in'
   - Ложные срабатывания при symbol = ""
   - Серьёзность: minor
   - Рекомендация: Заменить на `return symbol in string`

4. Метод: delete_symbol()
   - Нет обработки случая symbol = ""
   - При передаче пустого символа может вернуть пустую строку
   - Серьёзность: critical
   - Рекомендация: Добавить проверку if not symbol: return string
